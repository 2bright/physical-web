Physical Web

Walk up and use anything

View project on GitHub

What is this?

The Physical Web is an approach to unleash the core superpower of the web: interaction on demand. People should be able to walk up to any smart device - a vending machine, a poster, a toy, a bus stop, a rental car - and not have to download an app in order to use it. Everything should be just a tap away.

Why is this important?

The number of smart devices is going to explode. But today's assumption that each device requires it's own application just can't scale: no phone can hold every app you might need as you go through the day. We need a system that lets anyone interact with any device at any time. The Physical Web isn't about replacing native apps: it's about allowing interaction when native apps just aren't practical.

Why is Chrome sponsoring this?

The Physical Web must be an open standard that everyone can use. This can't be a product that is locked down by a single company. By creating a common web standard for all devices to build upon, a new range of services can just expect simple interaction with anyone to be possible.

How does this change things?

Once any smart device can have a web address, the entire overhead of an app seems a bit backward. The Physical Web approach unlocks tiny use cases that would never be normally be practical:

  • A bus stop tells you the next bus arrival;
  • Parking meters and vending machines all work the same way, letting you pay quickly and easily;
  • Any store, no matter how small, can offer an online experience when you walk in;
  • You walk up to a ZipCar, learn about it for the first time, decide to sign up, and immediately drive away.

These examples are about little bits of data and very simple interactivity. Sometimes it's the tiny ideas that can change the world.
